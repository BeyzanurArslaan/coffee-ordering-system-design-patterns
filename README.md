# ☕ Coffee Ordering System Using Design Patterns

## 📖 Overview

This project is a Java-based Coffee Ordering System developed as part of the **Software Design Patterns** course.

The application demonstrates the implementation of multiple **Gang of Four (GoF) Design Patterns** and **SOLID principles** to build a flexible, maintainable, and extensible software architecture.

---

## 🎯 Objectives

- Apply GoF Design Patterns in a real-world application
- Improve software maintainability and flexibility
- Follow SOLID design principles
- Develop a modular coffee ordering system

---

## 🛠 Technologies

- Java
- Maven
- IntelliJ IDEA
- Git

---

## ☕ Features

- Coffee ordering system
- Custom coffee creation
- Beverage customization
- Discount strategies
- Payment processing
- Order history
- Customer notifications
- Extensible architecture

---

## 🧩 Implemented Design Patterns

| Pattern | Purpose |
|---------|---------|
| Singleton | Single shared instance |
| Builder | Flexible object creation |
| Factory Method | Beverage creation |
| Decorator | Add coffee extras |
| Bridge | Separate abstraction and implementation |
| Composite | Group menu items |
| Strategy | Discount calculation |
| Template Method | Payment workflow |
| Memento | Order history |
| Chain of Responsibility | Order validation |
| Visitor | Sales reporting |
